# Layoffs CSV sources, tried in order. First one that returns a CSV with the
# required columns wins. Each line is a URL; '#' starts a comment.
#
# Override at workflow level by setting env LAYOFFS_CSV_URL — that takes priority.
#
# Required columns (any of the aliases in build_data.py is fine):
#   - a date column      (date / Date)
#   - a count column     (total_laid_off / Laid_Off / # Laid Off / ...)
#   - optional: industry (industry / Industry)

# Sources are fetched in order and MERGED (rows are concatenated, then
# de-duplicated on (company, date, num_laid_off)). The build is happy with any
# subset of these working — empty results are skipped.

# Active scraper, updated daily via GitHub Actions. Covers ~2025-01 onward.
